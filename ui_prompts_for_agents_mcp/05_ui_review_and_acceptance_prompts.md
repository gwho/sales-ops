# 05 — UI Review, Critique, and Acceptance Prompts

Use these prompts to make the UI less generic and more professional.

---

## Senior Designer Critique Prompt

```text
Act as a senior B2B SaaS product designer.

Critique this UI design for “Sales Admin Automation Toolkit”.

Evaluate:
- Does it look professional enough for a hiring manager?
- Does it look too generic or AI-generated?
- Is the workflow clear for a non-technical sales admin user?
- Are the tables readable?
- Are the status badges meaningful?
- Are there too many charts?
- Is the upload/export workflow obvious?
- Does it communicate business value quickly?

Then suggest improvements for:
- layout hierarchy
- typography
- spacing
- color usage
- table density
- empty states
- microcopy
- dashboard KPIs
- visual credibility
```

---

## Hiring Manager Demo Review Prompt

```text
Review this UI as if you are a hiring manager for a Sales Admin / Operations Executive role.

Does the demo quickly show that the candidate can help with:
- order processing accuracy
- inventory allocation
- payment follow-up
- Excel reporting
- supplier/customer follow-up
- workflow control

What parts are impressive?
What parts feel irrelevant or too technical?
What should be simplified before showing this in an interview?
```

---

## UI Acceptance Criteria Prompt

```text
Write UI acceptance criteria for Sales Admin Automation Toolkit.

Include:
- overview dashboard
- order upload flow
- validation error display
- inventory allocation table
- payment aging report
- report export center
- loading states
- empty states
- error states
- responsive behavior
- accessibility
- professional visual polish

The criteria should be specific enough for a coding agent to implement and verify.
```

---

## Final UI QA Checklist

```text
Before finalizing the UI, check:

1. Can a first-time user understand what the app does within 10 seconds?
2. Are all primary actions obvious?
3. Are status badges consistent across screens?
4. Are error messages business-readable?
5. Are tables readable without horizontal chaos?
6. Are charts meaningful and not decorative only?
7. Is there a clear workflow from upload to export?
8. Are sample data and fake business records realistic?
9. Does the app avoid overpromising as a full ERP?
10. Can the candidate explain each screen in an interview?
```
