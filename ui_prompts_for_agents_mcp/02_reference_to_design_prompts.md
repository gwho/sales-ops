# 02 — Screenshot / Reference-to-Design Prompts

Use these prompts when you paste UI screenshots into Figma, Figma Make, ChatGPT, Claude, Cursor, or another agent.

---

## Reference Adaptation Prompt

```text
Use these pasted dashboard screenshots as visual references only. Do not copy them exactly.

I want to adapt the best patterns into a unique UI for “Sales Admin Automation Toolkit”.

The app is for:
- order validation
- inventory allocation
- payment aging
- report export

Please identify:
1. which reference has the best dashboard layout
2. which reference has the best table design
3. which reference has the best KPI card style
4. which reference has the best sidebar/navigation
5. which reference has the best upload or workflow pattern
6. what visual elements should be avoided
7. how to combine the best ideas into one coherent design

Output a new wireframe plan for my project.
```

---

## UI Reference Selection Prompt

```text
Act as a senior B2B SaaS product designer.

I collected several dashboard UI screenshots for inspiration. Please help me choose which ones are suitable for a sales admin / operations automation tool.

Evaluate each reference by:
- professional credibility
- table readability
- workflow clarity
- suitability for non-technical users
- whether it looks too generic or AI-generated
- whether it supports upload, validation, status, and export workflows
- whether it can map to Next.js + Tailwind + shadcn/ui components

Then recommend which visual patterns to borrow and which to avoid.
```

---

## Screenshot-to-Wireframe Prompt

```text
Based on these dashboard screenshots, create a low-fidelity wireframe plan for Sales Admin Automation Toolkit.

Do not copy the screenshots exactly.

Create wireframes for:
1. Overview Dashboard
2. Order Validation
3. Inventory Allocation
4. Payment Aging
5. Reports & Export

For each screen, define:
- header area
- main sections
- cards
- tables
- action buttons
- status badges
- empty states
- responsive behavior
```
