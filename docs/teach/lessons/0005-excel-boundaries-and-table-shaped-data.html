Track 1 — Python, Data-as-Tables, and Contracts

Excel Boundaries and Table-Shaped Data

An Excel file is a table: rows and columns, some cells blank. Once Python reads it, that table becomes a DataFrame — the one new concept this lesson introduces, kept to the smallest useful slice.

Primary source

pandas — 10 Minutes to pandas (official quickstart). You only need the first third or so for Track 1 — rows, columns, and basic filtering.

A spreadsheet, described in Lesson 2's vocabulary

Picture sample_data/sample_orders.xlsx open in Excel: one row per order, one column per field (order_id, sku, quantity, and so on). In Lesson 2's terms, each row is naturally a dict — {"order_id": "SO-2026-001", "sku": "VIS-SLIT-001", "quantity": 3} — and the whole sheet is a list of those dicts, one per row. That's genuinely the mental model: a table is a list of same-shaped dicts.

DataFrame: pandas' name for "a table in memory"

pandas is a library for working with exactly this shape efficiently. When code reads an Excel file, it gets back a DataFrame — pandas' name for a table: rows, named columns, one type per column. Look at the function from Lesson 2 again:

def load_excel(file, sheet_name: int | str = 0) -> pd.DataFrame:
    """Load an Excel file (path or file-like object) into a DataFrame."""
    return pd.read_excel(file, sheet_name=sheet_name)

pd.read_excel(...) does the actual reading; the function just hands back whatever it produces. pd.DataFrame in the return-type hint is now legible: "this function returns a table."

Three operations cover everything Track 1 needs:

  • df.columns — the list of column names actually present, e.g. ["order_id", "sku", "quantity"].
  • df["sku"] — pull out one column's worth of values, top to bottom.
  • df[df["quantity"] == 0] — filter: keep only rows where a condition is true. Read the inner part first (df["quantity"] == 0, a yes/no answer per row), then the outer df[...] as "keep only the rows that said yes."

A missing column is a real, checkable question

Because df.columns is just a list of names, "does this file have the column I need?" becomes a plain membership check — no special Excel-reading logic required. That's exactly what this repo's validation function does:

def validate_required_columns(
    df: pd.DataFrame, required_columns: Iterable[str], file_label: str
) -> None:
    """Raise MissingColumnsError if any required column is absent from df."""
    missing = [column for column in required_columns if column not in df.columns]
    if missing:
        raise MissingColumnsError(file_label, missing)

[column for column in required_columns if column not in df.columns] is a list comprehension — a compact way to build a new list by filtering an existing one. Read it as an English sentence, right to left after "for": "for each column in required_columns, keep it if it's not in df.columns." The result, missing, is the list of required names that genuinely aren't present — one pass, every missing column found at once, not just the first.

A blank cell becomes None

An empty Excel cell doesn't vanish when read into a DataFrame — it becomes Python's None (pandas often displays it as NaN, "not a number," for numeric columns specifically, but the idea is the same: "no value here"). This matters because a business rule can check for it directly: df["sku"].isna() asks, per row, "is this cell blank?" — you'll see this exact pattern in Tutorial 02's discussion of intentionally imperfect sample rows.
