Track 3 — Presentation Without Leakage

Guided Trace: One Result, Into a Saved Workbook

No new concepts — this is rehearsal, exactly like Tracks 1 and 2's own capstones. One real PaymentAgingRow, traced through every stage of export_payment_aging_report, ending in the exact assertions a saved-and-reloaded workbook has to satisfy. Predict each answer yourself before revealing it; skipping straight to the reveal defeats the point.

What you're rehearsing

No new external resource — this lesson exercises Lessons 15 through 20 together, one stage at a time. If any single step feels impossible, that's a signal to revisit the specific lesson it depends on, not to push through.

The row you're tracing

Open tests/contract_fixtures.py and find:

PAYMENT_AGING_ROW_FIXTURE: PaymentAgingRow = {
    "invoice_id": "INV-2026-001",
    "customer_name": "Bright Medical Trading Ltd",
    "invoice_date": "2026-03-31",
    "due_date": "2026-04-30",
    "invoice_amount": 58000.00,
    "paid_amount": 0.0,
    "outstanding_amount": 58000.00,
    "days_overdue": 70,
    "aging_bucket": "61-90 Days",
    "follow_up_priority": "High",
    "suggested_action": "Call or email customer urgently",
}

This one row is already a finished, trusted fact — Lesson 15's rule means nothing you're about to trace will ever recompute "High", 70, or any other field here. Every step below only asks what happens to this already-decided data.

Step 1 — Does this row survive the Follow-up List filter?

Recall Lesson 20's allow-list: FOLLOW_UP_PRIORITIES = {"High", "Medium", "Low", "Watch"}. Predict: does PAYMENT_AGING_ROW_FIXTURE appear in the Follow-up List sheet?

Reveal

Yes. "High" in {"High", "Medium", "Low", "Watch"} is True, so this row is included in follow_up_rows and written to both Follow-up List and All Invoices with Aging — the second sheet is unfiltered, per Lesson 15's "pass everything through unchanged" rule.

Step 2 — Which sheets exist, and in what order?

Before checking, write down your own predicted list of sheet names for a payment aging report, in order — then open tests/contract_fixtures.py and find REPORT_MANIFEST_FIXTURES's payment_aging entry.

Reveal

["Aging Summary", "Follow-up List", "All Invoices with Aging", "Data Issues", "Draft Messages"] — five sheets, in this exact order, because that's the exact order export_payment_aging_report calls wb.create_sheet(...) (via _write_summary_sheet/_write_detail_sheet), and Lesson 17 taught that openpyxl preserves creation order, never alphabetizes.

Step 3 — Where does the manifest's sheet list actually come from?

Predict: if a future edit deleted the Data Issues sheet-writing call but forgot to update anything else, would the manifest still claim five sheets, or would it correctly shrink to four?

Reveal

It would correctly shrink to four. sheet_names = wb.sheetnames (Lesson 17) reads the real workbook's actual sheets, after every sheet-writing call has run — there is no separate, hand-maintained list anywhere for a developer to forget to update. Deleting a sheet call automatically and correctly changes what the manifest reports, with nothing left in an inconsistent state.

Step 4 — What happens to the draft message's long text?

Open tests/contract_fixtures.py and find DRAFT_MESSAGE_ROW_FIXTURE — its message_text field is several sentences long, with embedded blank lines. Predict: after this value passes through _safe_cell_value (Lesson 18) and gets written to the Draft Messages sheet, does the text itself change in any way?

Reveal

No — _safe_cell_value only intercepts None/NaN/NaT/pd.NA (Lesson 18). A real, non-blank string like this one passes through completely unmodified. What does change is presentation-only formatting applied afterward — the cell's column gets widened and wrapped so the multi-paragraph text is actually readable, a targeted step covered in Tutorial 06 Part 10, not this lesson's scope.

Step 5 — What if this row's priority had been something nobody anticipated?

Predict: if follow_up_priority were "Unexpected" instead of "High", would this row still appear in All Invoices with Aging? In Follow-up List?

Reveal

All Invoices with Aging: yes, unchanged — that sheet is never filtered, only formatted (Lesson 15). Follow-up List: no — "Unexpected" is not a member of FOLLOW_UP_PRIORITIES, so Lesson 20's allow-list excludes it by default, the safer of the two possible failure directions.

Retrieval check

After the whole export finishes, which single value does manifest["sheet_names"] and wb.sheetnames both trace back to?

A test asserts a cell's value right after _write_detail_sheet runs, without saving or reloading. Per Lesson 18, is that assertion fully trustworthy?
