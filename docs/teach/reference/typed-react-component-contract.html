Typed React Component Contract

A prop-typing cheat sheet for this repo's own components. Companion to Lesson 35; extends Track 1's Contracts & TypedDict Glossary from the Python side to the TSX side.

Inline type vs. named alias

ShapeWhen this repo uses itExample
Inline object typeA single well-known prop (most often children) with nothing beyond its type worth documenting.{ children: ReactNode } — app/(workspace)/layout.tsx, app/(public)/layout.tsx
Named type alias (XxxProps)Several fields, reuse across call sites, or a shape worth naming and documenting as a unit — a judgment call, not a field-count threshold.MetricCardProps, UploadPanelProps, StatusBadgeProps

What a trailing ? actually does

  • Widens the field's type to include undefined — it does not create a default.
  • A default only exists if the component's own destructuring supplies one: tone = "neutral", accept = ".xlsx".
  • An optional field with no destructured default is literally undefined inside the component when a caller omits it — the component's own logic must handle that case (e.g. UploadPanel's sampleFileName ? check).

The one rule that resolves "can this prop be a function?"

Not "are function props allowed" (they are) — the real question is which two environments the prop is crossing. A function prop is fine between two Client Components (same environment, the browser). A function prop cannot cross from a Server Component into a Client Component, because a server-side function value can't be serialized into something the browser can receive — see Lesson 33.

Quick check before adding a new prop to any component here

1. Does it need a name of its own (2+ fields) or is inline fine (children only)? 2. Should it be required or optional — and if optional, does it need a real default or is undefined a legitimate "not supplied" state the component already handles? 3. Is its type the narrowest honest description (a literal union of the real allowed values), or did it default to string/any out of convenience? 4. If it's a callback, confirm both ends of the prop live in the same Server/Client environment.
