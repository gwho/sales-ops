Serialization Boundary Checklist

A reusable checklist for any place Python data is about to become some other format's data — not just Excel. Companion to Lesson 18.

The general question

Any serialization boundary — a spreadsheet cell, a JSON field, a database column — should force one question, asked once, in one place: what values are legal here, and what happens to every value that isn't one of them? This repo's answer for Excel cells is _safe_cell_value. A future Track's answer for JSON API responses, or Postgres columns, will be a different function — but the question is identical.

Checklist for a new serialization boundary

  1. List every "nothing" representation that can actually reach this boundary — not just the ones you'd guess. This repo's list came from actually tracing where each value type is produced (TypedDict defaults vs. pandas upcasting), not from guessing in advance.
  2. Write one function, in one place, that every value passes through before crossing the boundary — never let two different call sites each write their own "is this blank" check.
  3. Narrow exception handling to exactly what the boundary can raise — this repo's pd.isna(value) call is wrapped in try/except (TypeError, ValueError), because it genuinely raises on some non-scalar inputs, never a bare except: that would also hide an unrelated bug.
  4. Test the artifact after it actually crosses the boundary — round-trip testing (Lesson 18). Never trust an in-memory object's state as proof of what the real, serialized artifact contains.

Round-trip testing, defined

Round-trip test: a test that exercises an artifact's real save/load path — not just the in-memory object still sitting in the process that created it. This repo's version: load_workbook(io.BytesIO(workbook_bytes)), never inspecting an unsaved Workbook directly. The concrete reason it matters here: a written empty string "" and a genuinely absent value both come back as None after an .xlsx save/reload — a fact no amount of staring at the in-memory object would ever reveal.
