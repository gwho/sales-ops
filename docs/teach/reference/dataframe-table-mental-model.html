DataFrame / Table Mental Model

The minimum pandas vocabulary this repo actually uses. Companion to Lesson 5.

The core equivalence

ExcelPython (Lesson 2)pandas
A whole worksheetA list of same-shaped dictsA DataFrame
One rowOne dict, e.g. {"sku": "A", "quantity": 3}One row of the DataFrame
One column headerA dict key, shared across every row's dictOne entry in df.columns
An empty cell—None (often shown as NaN for numbers)

Reading a file in

df = pd.read_excel(path)          # returns a DataFrame
df.columns                        # -> Index(['order_id', 'sku', 'quantity', ...])
df["sku"]                         # -> the sku column, top to bottom
df.iloc[0]                        # -> the first row, as a Series (dict-like)

Building a DataFrame from rows you already have

rows = [
    {"sku": "VIS-SLIT-001", "product_name": "Slit Lamp Examination Unit", "active": True},
    {"sku": "VIS-LOUPE-004", "product_name": "Surgical Loupe Set (Legacy)", "active": False},
]
df = pd.DataFrame(rows)

This is exactly the pattern src/sample_data.py's generate_*() functions use — build a plain list of dicts, then hand the whole list to pd.DataFrame(...) once at the end.

Checking for a required column

missing = [column for column in required_columns if column not in df.columns]

df.columns behaves like a list of names — in / not in checks membership exactly like it would on a plain Python list.

Filtering: read the inner part first

df[df["quantity"] == 0]
PieceMeaning
df["quantity"] == 0A yes/no answer, computed per row (a "boolean mask").
df[ ... ]"Keep only the rows where the inner expression said yes."

Combine conditions with & (and) / | (or), each side in its own parentheses:

orders_df[orders_df["sku"].notna() & ~orders_df["sku"].isin(known_skus)]

Read as: "rows where sku is not blank, and (~ = not) sku is not among the known SKUs."

Checking for blank cells

df["sku"].isna()      # True where the cell is blank/None, per row
df["sku"].notna()     # the opposite — True where a real value is present

Counting and naming, not just detecting

duplicate_mask = df["order_id"].duplicated(keep=False)
duplicated_ids = set(df.loc[duplicate_mask, "order_id"])
# duplicated_ids might be: {"SO-2026-010"}
# duplicate_mask.sum() counts how many rows the mask marked True

df.loc[mask, "column"] is "give me just that column, but only for the rows the mask kept" — combines filtering and column-selection in one step.
