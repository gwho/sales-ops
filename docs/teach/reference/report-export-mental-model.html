The four-step shape, every report

Every export_*_report function — order validation, inventory allocation, payment aging — follows the identical sequence:

1. _new_workbook()                     — Workbook(), remove the default "Sheet"
2. _write_summary_sheet(...) /
   _write_detail_sheet(...) × N         — wb.create_sheet(name), in the exact
                                           order sheets must appear
3. _save_workbook_bytes(wb)             — io.BytesIO() → wb.save(buffer)
                                           → buffer.getvalue()   (never a disk path)
4. _build_manifest(..., wb.sheetnames)  — reads the REAL sheets just created

   returns (workbook_bytes: bytes, manifest: ReportManifest)

Understanding this shape once means understanding all three report types — they differ only in which sheets get written in step 2, never in the surrounding shape.
